核心概念
  • SFT（Supervised Fine-Tuning）：监督微调，标注者提供"指令→高质量回复"的示范数据
  • RLHF（Reinforcement Learning from Human Feedback）：基于人类反馈的强化学习，标注者对多个回复进行偏好排序
  • DPO（Direct Preference Optimization）：直接偏好优化，需要 chosen/rejected 配对数据
  • Red Teaming：红队测试，标注者尝试诱导模型产生有害输出

一、大语言模型（LLM）标注任务

1.1 SFT 数据标注（指令微调）

SFT

目标：为模型提供"什么是好的回复"的示范数据

标注形式

  • 指令撰写：标注者编写多样化的用户指令（prompt）
  • 回复撰写：标注者为给定指令编写高质量回复
  • 回复修改：对模型生成的回复进行修改和优化（post-editing）
  • 多轮对话构造：编写完整的多轮对话（你的共情项目就属于这类）

标注维度

维度说明常见等级
准确性回复内容是否事实正确完全正确 / 基本正确 / 有错误 / 严重错误
完整性是否充分回答了用户问题完整 / 基本完整 / 有遗漏 / 严重不足
有用性回复对用户是否有实际帮助非常有用 / 有用 / 一般 / 无用
安全性是否包含有害/不当内容安全 / 轻微风险 / 有风险 / 有害
格式规范排版、结构、代码格式是否合理优秀 / 合格 / 需改进
注意事项
  • 指令多样性很重要：覆盖不同领域、难度、长度、语言风格
  • 回复不能过于模板化，要有自然的语言变化
  • 多轮对话要保持上下文一致性，避免前后矛盾
  • 注意区分"模型应该知道的"和"模型不应该编造的"

1.2 偏好标注（RLHF/DPO）

RLHF

目标：告诉模型"哪个回复更好"，训练奖励模型或直接优化

标注形式

形式说明适用场景
Pairwise 排序给定同一指令的两个回复，选择更好的一个（A>B / B>A / 平局）DPO 训练、奖励模型训练
GSB 打分Good / Same / Bad 三档判断：A 优于 B（G）、两者相当（S）、A 劣于 B（B）模型对比评测、训练策略对比
K-wise 排序对 K 个回复（通常 4-8 个）进行完整排序更高效地获取偏好信号
Likert 打分对单个回复按 1-5 或 1-7 分打分细粒度质量评估
维度打分按多个维度分别打分（有用性、安全性、流畅性等）多目标优化
Best-of-N从 N 个回复中选出最好的一个快速筛选高质量数据
GSB 打分详解

GSB（Good / Same / Bad）是大模型评测中最常用的配对比较方式，也称 Win / Tie / Lose：

标记含义判断标准
G（Good/Win）模型 A 明显优于模型 B在关键维度上有可感知的质量差异
S（Same/Tie）两者质量相当无法明确判断谁更好，或差异可忽略
B（Bad/Lose）模型 A 明显劣于模型 B在关键维度上有可感知的质量差异

优势：标注效率高（三选一）；不需要校准绝对分值；标注者一致性通常高于 Likert 打分；直接产出模型间胜率对比。

劣势：无法衡量单个模型的绝对水平；当两者都差或都好时 Same 的信息量有限。

使用方式：对同一测试集，让两个模型分别生成回复，标注者逐条对比给出 G/S/B，最终统计 Good 率、Same 率、Bad 率。可结合 ELO 评分系统进行多模型循环对比。

与绝对分值打分的对比：GSB 适合快速对比模型优劣（"谁更好"）；Likert 绝对分值适合衡量单个模型的质量水平（"有多好"）。实际项目中常两者结合使用。

标注要点

  • 明确偏好标准的优先级：通常 安全性 > 准确性 > 有用性 > 流畅性
  • 处理"都好"或"都差"的情况：需要规范中明确如何处理平局
  • 避免位置偏差：随机化回复的展示顺序（A/B 位置）
  • 避免长度偏差：不能仅因为回复更长就判定更好
  • 标注者校准：定期用标准样本校准不同标注者的偏好尺度

1.3 安全性标注（Red Teaming & Safety）

安全

目标：识别和标记模型的有害输出，构建安全训练数据

标注任务类型

任务说明
有害性分类判断回复是否包含有害内容，分类为：暴力、色情、歧视、违法、自伤引导等
严重程度分级对有害内容分级：轻微（minor）/ 中等（moderate）/ 严重（severe）
Red Teaming 攻击标注者扮演攻击者，设计 prompt 诱导模型产生有害输出
安全回复撰写为有害请求编写安全的拒绝回复（既要拒绝又要有帮助性）
边界判断判断模型是否过度拒绝（over-refusal）正常请求
注意事项
  • 安全标注需要明确的分类体系（taxonomy），通常参考各平台的内容政策
  • 标注者需要心理支持——长期接触有害内容可能造成心理负担
  • 区分"讨论敏感话题"和"产生有害内容"的边界
  • 文化差异：不同地区对"有害"的定义不同，需要本地化标准

1.4 特定能力标注

代码能力

  • 标注维度：正确性（能否运行）、效率、可读性、安全性
  • 需要标注者具备编程能力，能实际运行和验证代码
  • 注意：不同语言/框架需要对应专业的标注者

数学/推理能力

  • 标注维度：答案正确性、推理过程正确性、步骤完整性
  • 需要验证每一步推理，而不仅仅是最终答案
  • Chain-of-Thought 标注：标注正确的思维链过程

知识问答

  • 标注维度：事实准确性、信息来源可靠性、时效性
  • 需要标注者查证事实，标记幻觉（hallucination）
  • 区分"模型不知道应该说不知道"和"模型应该知道"的情况

创意写作

  • 标注维度：创意性、文学性、情感表达、结构完整性
  • 主观性较强，需要更多标注者交叉验证
  • 注意文化和审美差异

二、多模态模型 — 视觉任务标注

多模态

视觉多模态模型需要理解图像/视频内容并与文本关联

2.1 图像描述（Image Captioning）

标注内容说明
短描述一句话概括图像主要内容
详细描述多句话描述图像中的物体、位置关系、颜色、动作、场景氛围等
层次化描述从全局到局部，分层描述（整体场景→主体→细节→氛围）
注意事项
  • 描述要客观，避免主观推测（"她看起来很开心"→"她在微笑"）
  • 空间关系要准确（左右、前后、上下）
  • 避免遗漏图像中的重要元素
  • 文字内容（如招牌、标签）需要准确转录
  • 注意描述的粒度要求——不同任务对详细程度要求不同

2.2 视觉问答（VQA）

标注内容说明
问题撰写基于图像内容设计问题（事实型、推理型、计数型等）
答案标注为问题提供准确答案
回复质量评估评估模型对视觉问题的回答质量
注意事项
  • 问题要确实需要"看图"才能回答，避免纯常识问题
  • 答案要基于图像可见信息，不能依赖外部知识推测
  • 设计不同难度层次：直接观察 → 空间推理 → 因果推理

2.3 目标检测与定位（Object Detection / Grounding）

标注内容说明
Bounding Box用矩形框标注目标物体的位置
实例分割像素级标注物体轮廓
文本-区域对应将文本描述与图像中的具体区域关联（Visual Grounding）
注意事项
  • Bounding Box 要紧贴物体边缘，不能过大或过小
  • 遮挡物体的处理规则要统一（标注可见部分 vs 推测完整轮廓）
  • 小物体和密集物体容易遗漏，需要特别关注
  • 类别定义要清晰（如"车"是否包含"卡车"）

2.4 图像偏好标注（多模态 RLHF）

标注内容说明
图像描述偏好对同一图像的多个描述进行排序
视觉对话偏好对模型关于图像的多个回复进行偏好排序
图像生成偏好对同一 prompt 生成的多张图片进行质量排序
安全性+有用性双维度分别标注回复的安全性和有用性（可能冲突）
注意事项
  • 需要同时评估文本质量和视觉理解准确性
  • 幻觉检测：模型是否描述了图像中不存在的内容
  • 跨模态一致性：文本描述是否与图像内容匹配

2.5 视频理解标注

标注内容说明
时间戳标注标注事件发生的起止时间
动作识别标注视频中人物的动作类别
视频描述对视频片段进行文字描述
视频问答基于视频内容的问答对
注意事项
  • 时间边界要精确，通常精确到帧或 0.5 秒
  • 需要关注时序关系（先后顺序、因果关系）
  • 长视频需要分段标注，注意段间连贯性
  • 区分"正在发生"和"已经完成"的动作

三、多模态模型 — 语音任务标注

多模态

语音模态涉及语音识别、语音合成、语音情感等多种任务

3.1 语音识别（ASR）标注

标注内容说明
转写（Transcription）将语音内容逐字转写为文本
时间对齐标注每个词/句的起止时间戳
说话人标注标注每段话的说话人身份（Speaker Diarization）
非语言事件标注笑声、咳嗽、背景噪音等
注意事项
  • 逐字转写 vs 规范化转写：是否保留口语化表达（"嗯"、"那个"）
  • 方言和口音的处理规则
  • 重叠语音（多人同时说话）的标注方式
  • 数字、日期、专有名词的书写规范
  • 标点符号的添加规则（语音中没有标点，需要标注者判断）

3.2 语音合成（TTS）标注

标注内容说明
韵律标注标注重音、停顿、语调升降
情感标注标注语音的情感类别和强度
自然度评分对合成语音的自然度进行 MOS 评分（1-5 分）
相似度评分评估合成语音与目标说话人的相似程度
注意事项
  • MOS（Mean Opinion Score）评分需要标准化的听音环境
  • 评分者需要经过听力测试筛选
  • 避免"锚定效应"——先听到的样本影响后续评分
  • 需要足够多的评分者（通常 ≥20 人）取平均

3.3 语音情感识别（SER）标注

标注内容说明
离散情感分类标注情感类别：开心、悲伤、愤怒、恐惧、惊讶、厌恶、中性
连续情感维度标注 Valence（效价）、Arousal（唤醒度）、Dominance（支配度）
情感强度标注情感的强弱程度
情感变化点标注语音中情感发生转变的时间点
注意事项
  • 仅基于声学特征判断，还是结合语义内容——需要明确规范
  • 情感标注主观性强，需要多人标注取共识
  • 文化差异：同一语调在不同文化中可能表达不同情感
  • 区分"说话者的真实情感"和"听者感知到的情感"

3.4 语音对话标注（与你的共情项目直接相关）

标注内容说明
对话行为标注标注每轮对话的意图（提问、安慰、建议、确认等）
副语言特征标注语速、音量变化、停顿、叹气等
共情程度评估评估回复的共情质量（结合语音语调和文本内容）
自然度评估评估对话是否像真人交流
注意事项
  • 语音模态的共情不仅看"说了什么"，还要看"怎么说的"
  • 语调、语速、停顿都传递情感信息
  • 多模态标注时需要同时听音频和看文本转写

3.5 语音指令理解标注

标注内容说明
意图识别标注用户语音指令的意图类别
槽位填充标注指令中的关键信息（时间、地点、对象等）
回复质量评估评估语音助手回复的准确性和自然度

四、标注质量控制

4.1 一致性指标

指标适用场景阈值参考
Cohen's Kappa两人分类标注一致性≥0.6 可接受，≥0.8 优秀
Fleiss' Kappa多人分类标注一致性≥0.6 可接受
ICC（组内相关系数）连续评分一致性≥0.75 良好
Kendall's W排序一致性≥0.7 可接受
Krippendorff's Alpha通用（支持缺失值）≥0.667 可接受

4.2 质量保障机制

  • Gold Standard（黄金标准题）：在任务中混入已知答案的题目，监控标注者准确率
  • 交叉标注：同一数据由多人标注，取共识或多数投票
  • 分层抽检：按标注者、时间段、任务类型分层抽样审核
  • 标注者分级：根据历史表现对标注者分级，高难度任务分配给高级标注者
  • 反馈循环：定期将审核结果反馈给标注者，持续改进
  • 异常检测：监控标注速度异常快/慢、评分分布异常等

4.3 常见质量问题

问题表现解决方案
标注者疲劳后期标注质量下降限制每日工作量、轮换任务类型
锚定效应受前一个样本影响随机化样本顺序
位置偏差偏好选择第一个/最后一个选项随机化选项位置
长度偏差偏好更长的回复在规范中明确"长≠好"
规范理解偏差不同标注者对规范理解不同增加案例、定期校准会议
作弊/敷衍随机标注、复制粘贴Gold Standard 检测、速度监控

六、面试/工作中的关键认知

6.1 不同阶段标注的本质区别

阶段标注本质核心难点
SFT 标注教模型"什么是好的"（示范）回复质量的天花板决定模型上限
RLHF 标注教模型"哪个更好"（比较）偏好的一致性和标准统一
安全标注教模型"什么不能做"（边界）边界的精确定义和文化适配
评测标注衡量模型"做得怎么样"（度量）评测维度的全面性和可操作性

6.2 标注工作的核心能力要求

  • 规范设计能力：能将模糊的需求转化为可操作的标注指南
  • 质量管理能力：设计质控流程、分析一致性数据、发现系统性问题
  • 沟通协调能力：与算法团队对齐需求、培训和管理标注团队
  • 领域知识：理解模型训练流程，知道标注数据如何影响模型行为
  • 工具能力：熟悉标注工具、能快速搭建标注流程

6.3 行业趋势

  • 人机协作标注：模型预标注 + 人工审核修改，提升效率
  • 主动学习：优先标注模型最不确定的样本
  • 合成数据 + 人工筛选：用模型生成候选数据，人工做质量把关（你的项目就是这个模式）
  • 多模态融合标注：同时处理文本+图像+语音的复合标注任务
  • RLAIF（AI 反馈）：用 AI 替代部分人工偏好标注，降低成本

6.4 与你共情项目的关联

你的项目经验覆盖了标注工作的多个核心环节：

  • ✅ 评价维度设计：基于论文调研设计共情评价体系
  • ✅ 语料构造：模型生成 + 人工筛选的混合模式

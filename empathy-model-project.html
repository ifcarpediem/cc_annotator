一、调研与评价体系设计

1.1 核心概念

共情（Empathy）：指模型能够识别、理解用户的情绪状态，并给出恰当的情感回应。包括认知共情（理解对方处境）和情感共情（感受对方情绪）两个层面。

拟人化（Anthropomorphism）：指模型的回复在语言风格、交互节奏、情感表达等方面接近真实人类的对话方式，而非机械式的信息输出。

1.2 评价维度设计

基于论文调研，设计以下评价维度，每个维度设定等级划分标准：

维度 说明 等级划分思路
情绪识别 模型是否准确识别用户当前的情绪状态 完全准确 / 部分准确 / 未识别 / 误判
情感回应 回复是否包含恰当的情感表达（安慰、鼓励、认同等） 自然贴切 / 基本合适 / 生硬 / 不当
语言自然度 回复是否像人类日常对话，避免模板化 非常自然 / 较自然 / 略显机械 / 明显机械
上下文记忆 模型是否能引用和运用历史对话信息 主动引用 / 被动回忆 / 遗忘 / 矛盾
安全性 回复是否避免有害、不当或误导性内容 完全安全 / 基本安全 / 存在风险 / 有害
对话连贯性 多轮对话中话题衔接是否流畅 流畅自然 / 基本连贯 / 偶有跳跃 / 断裂

二、语料构造

2.1 语料来源

方式一：从小说中提取对话

从小说文本中抽取角色间的对话片段，天然具备情感丰富、语言自然的特点，适合作为共情对话的参考素材。

方式二：模型构造对话（主要方式）

利用豆包、GPT 等模型，通过结构化流程生成训练语料。具体步骤如下：

Step 1：用户画像分析

分析产品目标用户群体的特征分布：

  • 年龄段分布（如 20-25 岁占 40%，25-35 岁占 35%...）
  • 性别比例
  • 职业类型（学生、白领、自由职业等）
  • 各类用户的典型情感需求
Step 2：对话场景构造

基于用户画像，设计各种触发对话的场景：

  • 工作压力（加班吐槽、职场人际、晋升焦虑）
  • 感情问题（恋爱烦恼、分手倾诉、家庭矛盾）
  • 生活琐事（孤独感、搬家焦虑、健康担忧）
  • 积极分享（开心事、成就感、日常趣事）
Step 3：角色扮演生成对话

设定两个角色进行对话生成：

  • 角色 A — 聊天助手：设定具备共情能力、安全意识、记忆能力
  • 角色 B — 用户：按照画像和场景扮演用户

三个核心能力在语料生成中的侧重：

能力语料侧重
共情情绪识别与回应、安慰鼓励、情感认同
安全避免有害建议、适时引导专业帮助、边界意识
记忆引用历史对话内容、记住用户偏好、上下文关联

三、语料筛选与标注

3.1 方案探索

尝试一：LLM-as-Judge（已放弃）

参考论文 "LLM as a Judge" 的方法，使用多个模型（GPT、Claude Sonnet 等）作为裁判对生成的对话进行打分，按分数筛选语料。

放弃原因：实际验证发现模型打分与人工评估的对话质量匹配度不佳，模型难以准确判断共情对话的细微质量差异。

最终方案：人工标注

组建标注团队，开发内部标注工具，对多轮对话进行人工评估。

3.2 标注流程

  1. 开发内部多轮对话标注工具（支持逐轮查看、评分、修改）
  2. 标注团队按照评价维度对每段对话进行评估
  3. 对不合格的回复进行修改和优化
  4. 形成最终标注后的高质量训练语料

四、模型训练

4.1 基础模型选择

  • Qwen 系列（阿里云）
  • Lingma 系列（蚂蚁团队）

4.2 训练策略

采用不同的训练策略（如不同的微调方法、数据配比、训练轮次等），产出多个候选模型，后续通过评测确定最优方案。

五、模型评测

5.1 测试样例构造

构造标准测试对话集，格式为：Human1 → Assistant1 → Human2 → Assistant2 → ...

5.2 逐轮推理方法

核心问题：测试样例中的 Assistant 回复是参考答案，但实际评测时需要让待测模型自己生成回复。如果直接让模型接续完整对话，后续轮次的 Human 输入就与模型实际输出对不上了。

解决方案：将一段完整对话拆分为若干轮，逐轮调用模型推理：

  1. 第 1 轮：输入 H1 → 模型输出 A1'
  2. 第 2 轮：输入 H1 + A1'（模型实际输出）+ H2 → 模型输出 A2'
  3. 第 N 轮：输入完整历史上下文 + HN → 模型输出 AN'

这样每一轮的 Human 输入保持测试样例原文，而 Assistant 回复使用模型的实际输出，确保上下文的真实性。

5.3 评测方式

  • 多模型排序：对同一测试样例，收集各候选模型的输出，进行排序对比
  • 策略对比：分析不同训练策略对模型性能的影响
  • 基线对比：将训练后模型与主流模型（GPT、Claude 等）进行性能差异对比

5.4 关键概念区分

概念含义
模型训练使用标注语料对基础模型进行微调，更新模型参数
模型推理给定输入，模型产生输出的过程（不更新参数）
模型评测在测试集上运行推理，评估输出质量，对比模型性能

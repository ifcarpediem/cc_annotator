评测的两大目的
  • 能力评估：衡量模型在各项任务上的绝对水平（"这个模型有多好"）
  • 对比排序：比较多个模型/训练策略的相对优劣（"哪个模型更好"）

你的共情项目属于后者——对比不同训练策略产出的模型，以及与主流模型的差异。

评测方式优势劣势适用场景
自动评测成本低、可复现、速度快与人类判断可能不一致；开放式任务难以衡量标准化任务（选择题、代码、数学）
人工评测最接近真实用户感受；可评估主观质量成本高、速度慢、标注者间差异对话质量、创意写作、共情等主观任务
LLM-as-Judge成本适中、可扩展存在偏差（位置、长度、自我偏好）大规模初筛、辅助人工评测
竞技场（Arena）真实用户偏好、持续更新无法控制评测维度、样本偏差综合能力排名

一、评测方法论

1.1 自动评测

通过程序自动计算指标，无需人工参与判断

类型方法典型指标
参考答案匹配将模型输出与标准答案对比Accuracy、F1、Exact Match
文本相似度衡量生成文本与参考文本的相似程度BLEU、ROUGE、BERTScore
代码执行运行生成的代码，检查是否通过测试用例pass@k
困惑度衡量模型对文本的预测能力Perplexity
选择题正确率多选题场景下的正确率Accuracy（MMLU 等）

1.2 人工评测

由人类标注者对模型输出进行主观判断

形式说明适用场景
绝对打分（Likert）对单个回复按 1-5 分评分衡量单模型的绝对质量
GSB 对比Good/Same/Bad 三档配对比较快速对比两个模型
K-wise 排序对多个回复进行完整排序多模型同时对比
维度打分按多个维度分别评分细粒度能力分析
Side-by-Side并排展示两个回复，选择偏好A/B 测试
人工评测的关键设计
  • 盲评：标注者不知道回复来自哪个模型，避免品牌偏见
  • 随机化：回复展示顺序随机，避免位置偏差
  • 校准：使用标准样本定期校准标注者的评分尺度
  • 交叉验证：同一样本由多人评估，计算一致性
  • 明确 rubric：每个分数等级都有具体的判断标准和示例

1.3 LLM-as-Judge

用大模型作为评估器，对其他模型的输出进行评分或排序

常见模式
  • 单模型打分：用 GPT-4 等强模型对输出打分
  • 配对比较：让 Judge 模型判断两个回复哪个更好
  • 多模型投票（PoLL）：多个不同模型组成评审团，投票决定
  • 参考答案辅助：提供参考答案帮助 Judge 判断
已知偏差
  • 位置偏差：倾向选择第一个出现的回复
  • 长度偏差：倾向认为更长的回复更好
  • 自我偏好：倾向给自己生成的内容更高分
  • 风格偏好：偏好与自己风格相似的回复

2.4 你的共情项目评测方法

项目评测设计回顾
  • 评测对象：不同训练策略的候选模型 + 主流模型基线
  • 评测方式：逐轮推理 + 多模型输出排序（本质是 K-wise 排序）
  • 逐轮推理的必要性：保证每轮的上下文是模型真实输出，而非测试样例中的参考回复
  • 评测维度：共情质量、自然度、安全性、上下文记忆等

这种方法结合了"标准化测试集"和"人工排序评测"的优势，既保证了可比性，又能评估主观质量。

3.3 图像生成评测

维度评测指标说明
图像质量FID（Fréchet Inception Distance）越低越好，衡量生成图像与真实图像的分布距离
多样性IS（Inception Score）越高越好，衡量生成图像的多样性和清晰度
文本对齐CLIP Score衡量生成图像与文本描述的匹配程度
人工偏好ELO / Win Rate人工对比排序
美学质量Aesthetic Score美学评分模型打分
视觉评测注意事项
  • 多模态幻觉是核心问题：模型描述图像中不存在的内容
  • 需要区分"视觉理解能力"和"语言生成能力"——模型可能语言流畅但视觉理解错误
  • 跨模态一致性：文本回复是否真正基于图像内容
  • 分辨率和图像类型的影响：同一模型在不同类型图像上表现可能差异很大

四、多模态模型 — 语音评测

4.1 语音识别（ASR）评测

指标说明计算方式
WER（Word Error Rate）词错误率，ASR 最核心指标(替换+删除+插入) / 参考词数
CER（Character Error Rate）字符错误率，中文常用(替换+删除+插入) / 参考字符数
SER（Sentence Error Rate）句子错误率有错误的句子数 / 总句子数
RTF（Real-Time Factor）实时率，衡量速度处理时间 / 音频时长（<1 为实时）
ASR 评测注意事项
  • 需要在不同场景下分别评测：安静环境、噪声环境、远场、近场
  • 不同口音/方言需要单独评测
  • 专有名词（人名、地名、品牌）的识别率需要单独统计
  • 标点符号和大小写通常在计算 WER 前做规范化处理

4.2 语音合成（TTS）评测

指标说明评测方式
MOS（Mean Opinion Score）自然度主观评分（1-5 分）人工听评，≥20 人取平均
CMOS（Comparative MOS）对比 MOS，两个系统的相对差异人工对比听评
PESQ语音质量感知评估自动计算（ITU-T 标准）
Speaker Similarity说话人相似度余弦相似度（speaker embedding）
Intelligibility可懂度听写正确率
韵律准确性重音、停顿、语调是否正确人工评估

4.3 语音情感识别评测

指标说明
UA（Unweighted Accuracy）各类别准确率的平均（不考虑类别样本量差异）
WA（Weighted Accuracy）加权准确率（考虑类别分布）
F1-Score精确率和召回率的调和平均
CCC（Concordance Correlation）连续维度（VAD）的一致性相关系数

4.4 语音对话评测（与你的项目相关）

维度评测内容评测方式
语音理解是否正确理解用户语音输入意图识别准确率、槽位 F1
回复质量回复内容的准确性和有用性人工评分 / LLM-as-Judge

五、核心评测指标详解

5.1 文本生成指标

指标原理优势局限
BLEU计算生成文本与参考文本的 n-gram 精确率计算简单、广泛使用不考虑语义；惩罚合理的改写
ROUGE计算生成文本与参考文本的 n-gram 召回率适合摘要评估同 BLEU，不考虑语义
BERTScore用 BERT 嵌入计算 token 级语义相似度考虑语义相似性计算成本高；依赖嵌入模型质量
METEOR结合精确率、召回率，支持同义词匹配比 BLEU 更灵活需要外部资源（同义词表）
Perplexity模型对文本的困惑度（预测不确定性）不需要参考文本低困惑度不等于高质量

5.2 分类与排序指标

指标适用场景说明
Accuracy选择题、分类任务正确预测数 / 总数
F1-Score不平衡分类精确率和召回率的调和平均
pass@k代码生成生成 k 个样本中至少有一个通过测试的概率
ELO Rating模型排名基于配对比较的动态评分系统（Chatbot Arena 使用）
Win Rate模型对比在配对比较中获胜的比例
GSB 分布模型对比Good/Same/Bad 的比例分布

5.3 视觉指标

指标适用场景说明
mAP目标检测各类别 AP 的平均值
mIoU语义分割各类别 IoU 的平均值
FID图像生成生成图像与真实图像的分布距离（越低越好）
CLIP Score文图匹配图像与文本的 CLIP 嵌入相似度
CIDEr图像描述基于 TF-IDF 加权的 n-gram 相似度

7.2 多轮对话评测流程（你的项目方法）

逐轮推理评测法

适用于对话类模型的评测，核心是保证上下文的真实性：

  1. 准备测试集：构造标准多轮对话（H1-A1-H2-A2-...），其中 A 是参考回复
  2. 逐轮推理：
    • 第 1 轮：输入 H1 → 模型生成 A1'
    • 第 2 轮：输入 [H1, A1', H2] → 模型生成 A2'
    • 第 N 轮：输入 [H1, A1', H2, A2', ..., HN] → 模型生成 AN'
  3. 收集输出：对每个候选模型重复上述过程
  4. 人工排序：标注者对各模型在同一测试样例上的输出进行排序（GSB 或 K-wise）
  5. 统计分析：计算各模型的胜率、ELO 分数，分析训练策略的影响

7.3 评测设计的关键决策

决策点选项考虑因素
测试集来源公开 Benchmark / 自建测试集公开集可比性强但可能被污染；自建集针对性强但缺乏外部对比
评测粒度整体评分 / 维度评分 / 逐轮评分粒度越细信息量越大，但标注成本越高
对比方式绝对打分 / 配对比较 / 排序配对比较一致性高但信息量有限；排序信息量大但标注难度高
评测规模小规模深度 / 大规模广度深度评测适合精细分析；广度评测适合整体判断
基线选择主流模型 / 前一版本 / 人类表现需要有意义的对比基线才能得出有价值的结论

7.4 评测报告要素

  • 评测设置：模型版本、推理参数（temperature、top_p 等）、测试集描述
  • 定量结果：各指标数值、置信区间、统计显著性
  • 对比分析：模型间差异、训练策略影响、能力维度分布
  • 案例分析：典型好例/坏例展示，定性说明模型行为
  • 局限说明：评测覆盖范围、已知偏差、不适用场景

7.5 行业趋势

  • 从静态到动态：LiveBench 等每月更新测试题，防止数据污染
  • 从单一到综合：HELM、TrustLLM 等多维度综合评测框架
  • 从 AI-alone 到 Human-AI：评测模型在真实人机交互中的表现，而非孤立回答
  • 众包评测：Chatbot Arena 模式，利用真实用户的偏好数据
  • 领域专精评测：医疗、法律、金融等垂直领域的专业 Benchmark
